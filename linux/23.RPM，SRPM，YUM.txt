厂商在固定的系统上编译好软件，然后发布将这个编译好的软件文件。
该文件还包含预先检测系统和依赖软件的脚本。
用户获取这个文件后，执行特定的命令，该软件文件就会按照内部的脚本来检测相关的前驱软件是否存在，
若安装的环境符合需求，那就会开始安装，安装完成后还会将该软件的信息写入软件管理机制，以完成未来可以进行升级，删除等操作。

distribution代表	软件管理机制(命令)	在线升级机制(命令)
Red Hat/Fedora		RPM（rpm,rpmbuild）	YUM（yum）
Debian/Ubuntu		DPKG（dpkg）		APT（apt-get）

RPM：RedHat Package Manager
	1.由于已经编译完成并且打包完毕，所以软件传输与安装上很方便（不需要再重新编译）
	2.由于软件的信息都已经记录在linux主机的数据库上，很方便查询，升级与反安装
    缺点：依赖
	  树形依赖，环形依赖，库文件依赖
    包全名和包名
	  操作没有安装过的软件包(安装，升级)，使用包全名，进入安装目录或使用绝对路径
	  操作已经安装过的软件包(卸载，查询)，使用包名，不需要进入安装目录，是搜索/var/lib/rpm/中的数据库
    命名：rp-pppoe   -	3.1  -	  5	     .i386	 .rpm
	  软件名称	版本	发布次数  适合的硬件平台 扩展名	  
    安装命令：rpm -ivh 包全名
		-i：install的意思
		-v：查看详细的安装信息画面
		-h：以安装信息栏显示安装进度
    升级命令：rpm -Uvh 包全名
		-U：update升级
    卸载命令：rpm -e 包名
		-e：erase卸载
    查询命令：rpm -q 包名
		-q：query查询是否安装
	      rpm -qa 包名
		-a：all查询所有安装过的prm包
	      rpm -ql 包名
		-l：list列出rpm包中文件的安装位置
	      rpm -qf 系统文件名
		-f：file查询系统文件属于哪个rpm包
    校验命令：rpm -V 包名
		-V：verify校验文件内容是否被改变


SRPM：Source RPM，这个RPM文件里含有源代码，扩展名为*.src.rpm
	1.先将该软件已RPM管理的方式编译，此时SRPM会被编译成为RPM文件
	2.然后将编译完成的RPM文件安装到linux系统中
    命令：rpmbuild

YUM：	RPM软件文件会有属性依赖问题，RPM软件内部会有记录依赖属性的清单，
	在安装软件需求的时候，将该清单与系统已安装软件比较，没安装的依赖软件会一口气同时安装起来。
	CentOS将发布的软件放置到YUM服务器(repository)内，并将依赖属性分析后记录成该软件相关性的清单列表。
	客户需要安装软件时，先向yum服务器下载清单列表，再与本机RPM数据库已存软件数据相比较，最后一口气安装具有依赖属性的软件。

yum源文件：
	生效文件为 /etc/yum.repos.d/CentOS-Base.repo
	国内也有yum源，比如163

光盘搭建yum源：
	1.挂载光盘，光盘中有rpm包
	2.使网络yum源失效，修改CentOS-Base.repo的后缀名即可
	3.修改/etc/yum.repos.d/CentOS-Media.repo中的配置

服务器使用最小化安装，用什么软件安装什么，尽量不卸载。
yum命令：
	yum list -- 查询所有可用软件包列表
	yum search 关键字 -- 搜索服务器上所有和关键字相关的包
	yum -y install 包名
	    -y 自动回答yes
	yum -y update 包名 -- 升级会重启服务，能不更新就不更新
	yum -y remove 包名
	yum grouplist 
	yum groupinstall 软件组名
	yum groupremove 软件组名

RPM和YUM
	RPM是rpm包的本地安装
	YUM是rpm包的在线安装，不能查询和校验
